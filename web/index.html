Settings

Preferences below are stored on the EvuProxy host as ui-preferences.json next to the main config (not in WireGuard YAML). The API token is on the sidebar page “API token” and stays in the browser only.

Peer tunnel subnet

IPv4 CIDR used to pick the next free tunnel IP when you add a peer. Should match the WireGuard tunnel network. Server default is 10.100.0.0/24 when unset.

WireGuard server endpoint

Public address peers use to reach this VPN (host:port or hostname:port). Pre-filled in the peer form; port can be omitted if it matches the server listen port after you refresh server details.

Advanced settings

Enables advanced options across this admin UI. Stored only in this browser (localStorage).

WireGuard profile

                

                  Server endpoint and keys below are used to build the YAML snippet and WireGuard config for this peer. Use narrow AllowedIPs (tunnel subnet only). Many setups save the [Interface] / [Peer] block as client.conf. New peers get a keypair automatically; you can clear and regenerate while adding.
                

                

                  
                  
                

                

                  
                

                

                

                  
                  

                    
                    
                  

                

                
Install on Linux (systemd)

                

                  Installs wireguard-tools, writes /etc/wireguard/, and enables wg-quick@…. Server public key and listen port load from the API when you change any field in this dialog. The command contains your private key — avoid shared terminals and shell history on untrusted hosts. The script is downloaded to a temp file, sha256sum is printed so you can verify it against SHA256SUMS on the GitHub Release (same tag or commit as the script URL) before sudo bash. Set window.EVUPROXY_PEER_INSTALL_SCRIPT_URL to a release-pinned raw URL for a stable trust anchor.

Geoblocking

        

          When enabled, nftables restrict public forwarding and input_allows ports using
          downloaded country zones. Choose whether the country list is an allow or block
          list. Stored in geo in the main config. After saving, run
          Update geo lists on this page or from Overview, then Apply on Pending changes.

Logs

        

          Lines from nftables log rules with prefixes
          evuproxy-geo-block (geoblocking drops) and
          evuproxy-forward-drop (forward-chain drops). The API reads
          journalctl (newest first) or falls back to dmesg.
